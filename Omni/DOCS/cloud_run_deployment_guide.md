# OmniSense AI: Cloud Run Deployment Guide

This guide provides step-by-step instructions for deploying the OmniSense AI platform to Google Cloud Platform (GCP) using Cloud Run.

## 1. Prerequisites
- Google Cloud Project: `project-017f83aa-1166-4bbf-aae`
- GCloud SDK installed and authenticated: `gcloud auth login`
- Docker installed locally.

## 2. Infrastructure Setup

### Enable Services
```bash
gcloud services enable \
    run.googleapis.com \
    containerregistry.googleapis.com \
    firestore.googleapis.com \
    redis.googleapis.com \
    secretmanager.googleapis.com \
    pubsub.googleapis.com
```

### Provision Resources
1. **Firestore**: Ensure a database named `omnisence-data` is created in Native mode.
2. **Memorystore**: Create a Redis instance named `omnisense` in `us-central1`.
3. **Cloud Storage**: Create a bucket named `omnisensebkt`.
4. **Secrets**: Add your `GEMINI_API_KEY` to Secret Manager.

## 3. Backend Deployment

### Build & Push
```bash
cd Omni/backend
gcloud builds submit --tag gcr.io/project-017f83aa-1166-4bbf-aae/omnisense-backend .
```

### Deploy to Cloud Run
```bash
gcloud run deploy omnisense-backend \
    --image gcr.io/project-017f83aa-1166-4bbf-aae/omnisense-backend \
    --platform managed \
    --region us-central1 \
    --allow-unauthenticated \
    --set-env-vars="Memorystore=YOUR_REDIS_IP,FIRESTORE_DATABASE_ID=omnisence-data,SNAPSHOT_BUCKET=omnisensebkt" \
    --update-secrets="GEMINI_API_KEY=GEMINI_API_KEY:latest"
```

## 4. Frontend Deployment

### Configuration
1. Update `Omni/frontend/.env` with your Backend URL:
   `VITE_API_URL=https://omnisense-backend-xyz.a.run.app`

### Build & Push
```bash
cd Omni/frontend
gcloud builds submit --tag gcr.io/project-017f83aa-1166-4bbf-aae/omnisense-frontend .
```

### Deploy to Cloud Run
```bash
gcloud run deploy omnisense-frontend \
    --image gcr.io/project-017f83aa-1166-4bbf-aae/omnisense-frontend \
    --platform managed \
    --region us-central1 \
    --allow-unauthenticated
```

## 5. Network Configuration (VPC Connector)
To connect Cloud Run to Memorystore (Redis), you **must** create a Serverless VPC Access connector:
```bash
gcloud compute networks vpc-access connectors create omnisense-connector \
    --region us-central1 \
    --range 10.8.0.0/28
```
Then, update your backend service to use this connector:
```bash
gcloud run services update omnisense-backend --vpc-connector omnisense-connector
```
